# 日本語学習ノート — ITエンジニア向け

ITエンジニアのための日本語学習記録。日本の職場環境で使われる実践的な言語を中心に、**N2/N3**レベル（時々N4）の内容をまとめています。

学習者：Falco（ベトナム人ITエンジニア）| 目標：日本企業での技術文書の読解・コミュニケーション

---

## フォルダ構成

```
開発者/
├── README.md          # リポジトリ全体の説明
├── documents/         # Jekyll / Just the Docs のサイト本体
│   ├── 毎日練習/          # 毎日自動生成される読解練習（N2/N3）
│   │   └── YYYY-MM-DD.md
│   │
│   ├── ニュース/          # 日本のIT系ニュース・記事の読解
│   │   └── YYYY-MM/
│   │       └── YYYY-MM-DD.md
│   │
│   ├── テクニカル/        # 技術用語：インフラ・CI/CD・DB・セキュリティ・アーキテクチャ
│   ├── クライド技術/      # Claude API・Claude Code・Agent SDK・プロンプトエンジニアリング
│   ├── AI時代/           # AI時代の開発者キャリア・ツール・日本のIT環境
│   │
│   ├── スキル/            # ビジネス日本語スキル（メール・Slack・報連相・敬語・コードレビュー）
│   ├── ビジネス日本語/    # BJT・進捗報告・依頼・社内コミュニケーション
│   ├── 文法トレーニング/  # 動詞活用・自動詞/他動詞・受身・使役・ビジネス文法
│   ├── 設計ドキュメント/  # 仕様書・設計書・要件定義書の読み書き・APIレビュー
│   ├── エラー・障害対応/  # 障害対応・ログ読解・インシデント報告・ポストモーテム
│   └── 面接・転職/        # 技術面接・経験説明・給与交渉・転職理由
│
├── collections/       # documents 外の横断復習コレクション
│   ├── 語彙/              # IT・ビジネス・生活の語彙フラッシュカード
│   └── 文法/              # IT・ビジネス・生活の文法/表現カード
│
├── .codex/            # Codex 用プロンプト・メモ・ローカルスキル
└── .github/           # GitHub Actions ワークフロー
```

命名ルール：日付ファイル名（`YYYY-MM-DD.md`）を使うのは `毎日練習` と `ニュース` だけです。その他の学習セクションは `NN_テーマ.md` の形式で、番号とテーマ名を使います。

## 横断復習コレクション

`collections/` は、`documents/` にある記事を横断して復習するためのフォルダです。

| フォルダ | 内容 |
|----------|------|
| [collections/語彙](./collections/語彙/) | `単語 -> 読み -> 意味メモ -> 使う文脈 -> 例文 -> 元記事` のカテゴリ別フラッシュカード |
| [collections/文法](./collections/文法/) | `文法/表現 -> 意味・使い方 -> 使う文脈 -> 例文 -> 元記事` のカテゴリ別フラッシュカード |

カテゴリは `IT`、`ビジネス`、`生活` の3つです。`documents/` のフォルダ名そのままではなく、実際に使う場面に合わせて分類します。

新しい記事を `documents/` に追加した場合は、記事内に `語彙リスト` と `文法・表現ポイント` を入れた上で、次のコマンドを実行してコレクションも更新します。

```powershell
powershell -ExecutionPolicy Bypass -File collections/generate-collections.ps1
```

生成後は、[語彙](./collections/語彙/) と [文法](./collections/文法/) のカテゴリ別ファイルを見て、例文が自然か確認します。
