## 📖 読解文（約500字）

本番環境で障害が発生した際、エンジニアには迅速な対応と正確な情報共有が求められる。日本のIT企業では、障害対応の流れが「検知→一次対応→原因調査→暫定対応→恒久対応→再発防止」という順序で標準化されていることが多い。各フェーズで適切な日本語表現を使いこなすことが、チームとの連携を円滑にする上で非常に重要だ。

まず「検知」の段階では、アラートを受けた直後にSlackや社内チャットへ第一報を投稿する。この時点では原因が分からなくても、「現在調査中」という状況を共有することで、関係者が不安なく待てる環境を作ることができる。続く「原因調査」では、ログやモニタリングツールを活用して根本原因（root cause）を特定する。

「暫定対応」とは、完全な修正は間に合わないが、サービスを早期に復旧させるための一時的な措置を指す。例えばリリースを切り戻したり、特定の機能を一時停止したりすることが該当する。そして「恒久対応」では、根本的な修正をコードや設定に加える。最後の「ポストモーテム」では、障害の全経緯を文書化し、再発防止策をチーム全体で共有することが求められる。

## ❓ 読解問題

1. 障害対応フローを順番に並べてください。
2. 「検知」の段階で原因がわからなくても第一報を投稿する理由は何ですか？
3. 「暫定対応」と「恒久対応」の違いを説明してください。

<details>
<summary>解答</summary>

1. 検知 → 一次対応 → 原因調査 → 暫定対応 → 恒久対応 → 再発防止。
2. 「現在調査中」という状況を共有することで、関係者が不安なく待てる環境を作るため。
3. 暫定対応は一時的にサービスを復旧させるための措置、恒久対応はコードや設定に根本的な修正を加えること。

</details>

## 📐 文法ポイント

### 1. 〜に際して（N2）
> 本番環境で障害が発生した**際**、エンジニアには迅速な対応が求められる

「〜に際して / 〜した際（に）」= "when ~" / "at the time of ~"（重要な場面・タイミング）  
例：デプロイに際して、必ずステージング環境でテストを行う。

### 2. 〜ことで（N3）
> 「現在調査中」と共有する**ことで**、関係者が不安なく待てる環境を作ることができる

「〜ことで」= "by doing ~"（手段→結果）  
例：ログを残すことで、原因特定が速くなる。

### 3. 〜とは〜を指す（N2）
> 「暫定対応」**とは**、サービスを早期に復旧させるための一時的な措置**を指す**

「〜とは〜を指す / 〜のことだ」= "~ refers to ~" / "~ means ~"（定義・説明）  
例：「技術的負債」とは、将来の開発速度を下げる設計上の問題を指す。
