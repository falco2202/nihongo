# 02 自動詞・他動詞トレーニング

## 目的

自動詞・他動詞は、ベトナム語話者が特に間違えやすいポイントです。日本語では、動作そのものだけでなく、「自然にそうなったのか」「誰かがそうしたのか」を動詞で分けることが多いです。

基本の見方は次の通りです。

| 種類 | 助詞 | 意味 | 例 |
|------|------|------|----|
| 自動詞 | が | もの・状態が自然に変わる | ドアが開く |
| 他動詞 | を | 誰かが対象に働きかける | ドアを開ける |

---

## よく使うペア

| 自動詞 | 他動詞 | 職場例文 |
|--------|--------|----------|
| 開く | 開ける | ファイルが開きません。ファイルを開けてください。 |
| 閉まる | 閉める | チケットが閉まりました。担当者がチケットを閉めました。 |
| 入る | 入れる | データがDBに入りました。データをDBに入れました。 |
| 出る | 出す | エラーが出ました。ログを出してください。 |
| 始まる | 始める | 会議が始まりました。会議を始めます。 |
| 終わる | 終える | 処理が終わりました。作業を終えました。 |
| 壊れる | 壊す | ビルドが壊れました。設定を壊してしまいました。 |
| 直る | 直す | バグが直りました。バグを直しました。 |
| 集まる | 集める | 情報が集まりました。情報を集めます。 |
| 決まる | 決める | 方針が決まりました。方針を決めました。 |
| 見つかる | 見つける | 原因が見つかりました。原因を見つけました。 |
| 変わる | 変える | 仕様が変わりました。仕様を変えました。 |

---

## 判断のコツ

| 質問 | 自動詞・他動詞 |
|------|----------------|
| 何が起きたかを言いたい | 自動詞 |
| 誰が何をしたかを言いたい | 他動詞 |
| 原因や責任をぼかしたい | 自動詞 |
| 作業内容を明確に言いたい | 他動詞 |

例:

- デプロイ後、エラーが出ました。
- 私が設定を変えたため、エラーが出ました。

1つ目は状況説明です。2つ目は作業者と原因を明確にしています。

---

## ITでよく使う自然な表現

| 自然な表現 | 避けたい表現 | メモ |
|------------|--------------|------|
| エラーが出る | エラーを出る | 「エラー」は自然に発生するものとして言う |
| ログを出す | ログが出す | 「ログを出力する」の意味 |
| サーバーが落ちる | サーバーを落ちる | 障害として落ちたなら自動詞 |
| サーバーを落とす | サーバーが落とす | 意図的に停止するなら他動詞 |
| 設定が変わる | 設定を変わる | 変化なら自動詞 |
| 設定を変える | 設定が変える | 誰かが変更するなら他動詞 |

## 練習A: 助詞を入れる

空欄に「が」または「を」を入れてください。

1. 仕様（　）変わりました。
2. 仕様（　）変えました。
3. 原因（　）見つかりました。
4. 原因（　）見つけました。
5. チケット（　）閉まりました。
6. チケット（　）閉めました。
7. エラー（　）出ました。
8. ログ（　）出しました。

1. 仕様が変わりました。

## 練習B: 自然な文に直す

次の文を自然な日本語に直してください。

1. エラーを出ました。
2. 私は原因が見つけました。
3. 設定を変わりました。
4. 会議を始まりました。
5. バグを直りました。

<details>
<summary>解答例</summary>

1. エラーが出ました。
2. 私は原因を見つけました。
3. 設定が変わりました。 / 設定を変えました。
4. 会議が始まりました。 / 会議を始めました。
5. バグが直りました。 / バグを直しました。

</details>

## 練習C: 報告文を作る

次の内容を、状況説明と作業説明の2文に分けてください。

例:

- 内容: setting changed
- 状況説明: 設定が変わりました。
- 作業説明: 設定を変えました。

1. bug fixed
2. root cause found
3. ticket closed
4. process finished
5. data inserted into DB

<summary>解答例</summary>

1. バグが直りました。 / バグを直しました。
2. 原因が見つかりました。 / 原因を見つけました。
3. チケットが閉まりました。 / チケットを閉めました。
4. 処理が終わりました。 / 作業を終えました。
5. データがDBに入りました。 / データをDBに入れました。

</details>
