## 📖 読解文（約500字）

クラウドインフラを設計する際に欠かせない概念の一つが「冗長化」である。冗長化とは、システムの一部が故障しても全体のサービスが停止しないよう、同じ機能を持つ構成要素を複数用意することを指す。これにより、単一障害点（SPOF）をなくし、システムの可用性を高めることができる。

AWSを例に取ると、複数のアベイラビリティゾーン（AZ）にサーバーを分散配置することで、一つのデータセンターで障害が発生しても他のAZが処理を引き継ぐ構成を実現できる。ロードバランサーを組み合わせることで、トラフィックを自動的に分散させ、特定のサーバーへの負荷集中を防ぐことも可能だ。

可用性の指標としてよく使われるのが「SLA（サービスレベル合意）」で、一般的に「99.9%」や「99.99%」という形で表現される。99.9%の場合、年間の許容ダウンタイムは約8.7時間となる。本番環境の設計では、目標とするSLAを明確にした上で、それを達成するために必要な冗長構成を決定する。この判断は、コストと可用性のトレードオフを考慮しながら行うことが求められる。

1. 「冗長化」とはどのような概念ですか？本文の説明を使って答えてください。
2. AWSで複数のAZにサーバーを配置する目的は何ですか？
3. SLA「99.9%」とは、年間でどのくらいのダウンタイムを許容することを意味しますか？

<details>
<summary>解答</summary>

1. システムの一部が故障しても全体が停止しないよう、同じ機能を持つ構成要素を複数用意すること。
2. 一つのデータセンターで障害が発生しても、他のAZが処理を引き継げるようにするため。
3. 年間約8.7時間のダウンタイムを許容すること。

## 📐 文法ポイント

### 1. 〜ことができる（N4/N3）
> 単一障害点をなくし、可用性を高める**ことができる**

「〜ことができる」= "can ~" / "to be able to ~"（可能）  
例：ロードバランサーを使うことで、負荷を分散することができる。

### 2. 〜た上で（N2）
> 目標とするSLAを明確にし**た上で**、冗長構成を決定する

「〜た上で」= "after doing ~" / "having done ~, then"（前提として先に完了する動作）  
例：要件を確認した上で、設計を始める。

### 3. 〜を考慮しながら（N2）
> コストと可用性のトレードオフ**を考慮しながら**行うことが求められる

「〜ながら」= "while ~ing"（同時進行）  
例：リスクを考慮しながら、最適な方法を選ぶ。
