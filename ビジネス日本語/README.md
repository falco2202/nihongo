## 📖 読解文（約500字）

日本の職場では「報連相（ほうれんそう）」という言葉がよく使われる。これは「報告・連絡・相談」の頭文字を組み合わせた造語で、社員が上司や同僚と情報を共有するための基本的な習慣を指す。IT企業においても、この概念はスクラム開発のデイリースタンドアップや、Slackでの進捗共有など、様々な形で取り入れられている。

「報告」とは、指示されたタスクの結果や進捗を上司に伝えることだ。重要なのは、結論から先に述べること。日本語では「PREP法（Point・Reason・Example・Point）」に近い構成が好まれる。「○○が完了しました。理由は△△です。具体的には〜」という順序で話すと、相手が理解しやすい。

「連絡」は、業務に関わる情報を関係者に共有することで、決定事項や変更点をタイムリーに伝えることが求められる。「相談」は、判断が必要な場面で上司や同僚の意見を求めることで、問題が小さいうちに動くことが大切とされる。この報連相を適切に行うことで、チーム全体の認識が揃い、手戻りや手違いを大幅に減らすことができる。

## ❓ 読解問題

1. 「報連相」はそれぞれ何の略ですか？
2. 「報告」を行う際に重要とされる話し方のポイントは何ですか？
3. 「相談」で大切とされていることは何ですか？

<details>
<summary>解答</summary>

1. 報告・連絡・相談の略。
2. 結論から先に述べること。
3. 問題が小さいうちに動くこと（早めに相談すること）。

</details>

## 📐 文法ポイント

### 1. 〜とされる（N2）
> 問題が小さいうちに動くことが大切**とされる**

「〜とされる」= "it is considered that ~" / "is regarded as ~"（一般的な認識・評価）  
例：このパターンはベストプラクティスとされている。

### 2. 〜うちに（N3）
> 問題が小さい**うちに**動くことが大切

「〜うちに」= "while ~" / "before ~ changes"（状態が変わる前に行動する）  
例：バグが広がらないうちに修正する。

### 3. 〜ことで（N3）
> 報連相を適切に行う**ことで**、手戻りや手違いを減らすことができる

「〜ことで」= "by doing ~"（手段・理由）  
例：ドキュメントを整備することで、引き継ぎがスムーズになる。
